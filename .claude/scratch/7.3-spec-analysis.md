# Spec Analysis: 7.3

## Algorithms to Implement

### Support Types (7.3.1-7.3.4)
**Spec Sections:** 7.3.1, 7.3.2, 7.3.3, 7.3.4

Each Support dictionary must have:
1. `supported` - boolean indicating if config is supported
2. `config` - the configuration used in determining support

**Types:**
- AudioDecoderSupport: { supported: boolean, config: AudioDecoderConfig }
- VideoDecoderSupport: { supported: boolean, config: VideoDecoderConfig }
- AudioEncoderSupport: { supported: boolean, config: AudioEncoderConfig }
- VideoEncoderSupport: { supported: boolean, config: VideoEncoderConfig }

**Error Conditions (spec-mandated):**
- None (types are return values, not inputs)

**Edge Cases (from spec):**
- supported: false should still have config field present
- config should be cloned (per 7.2)

## Inputs NOT in Test Requirements (Must Still Work)
- Config with all fields populated
- Config with only required fields
