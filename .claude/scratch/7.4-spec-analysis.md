# Spec Analysis: 7.4

## Algorithms to Implement

### Codec String Validation (7.4)
**Spec Section:** 7.4

A valid codec string MUST:
1. Be valid per the relevant codec specification
2. Describe a single codec
3. Be unambiguous about profile, level, and constraint bits

**Codec String Formats (from WebCodecs Codec Registry):**

Video:
- H.264/AVC: avc1.PPCCLL (e.g., avc1.42001E = Baseline 3.0)
- H.265/HEVC: hvc1.* or hev1.*
- VP9: vp09.PP.LL.DD (e.g., vp09.00.10.08)
- AV1: av01.P.LLT.DD (e.g., av01.0.04M.08)

Audio:
- AAC: mp4a.40.XX (e.g., mp4a.40.2 = AAC-LC)
- Opus: opus
- MP3: mp3
- FLAC: flac
- Vorbis: vorbis

**Error Conditions (spec-mandated):**
- Empty string → invalid
- Unknown codec → isConfigSupported returns false
- Malformed codec string → isConfigSupported returns false

**Edge Cases (from spec):**
- Case sensitivity varies by codec
- avc1 without profile/level may be handled implementation-specific

## Inputs NOT in Test Requirements (Must Still Work)
- Different H.264 profiles (42=Baseline, 4D=Main, 64=High)
- VP9 with different profiles/levels
- AV1 codec strings
